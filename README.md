## Online Retail Sales Database Design
A normalized SQL-based e-commerce database designed to manage products, customers, orders, payments, and inventory. It supports transactional integrity, automated stock updates, and sales reporting using joins, views, and triggers for real-world retail operations.

## Features
- Normalized database schema (3NF)
- Product, customer, order, and payment management
- Inventory stock tracking
- Automatic stock reduction using triggers
- Low-stock alert query
- Sales reports using JOIN queries
- Daily sales summary using SQL views

## Technologies Used
- MySQL 
- SQL (DDL, DML, Triggers, Views, Joins)
- dbdiagram.io (ER diagram design)

## Steps to Run
- Create a database in MySQL.
- Execute the table creation scripts (DDL).
- Insert sample data using provided INSERT queries.
- Run trigger scripts for inventory updates.
- Execute SELECT queries and views to generate reports.
